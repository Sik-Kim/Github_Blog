- 현재 머물고 있는 directory 안의 파일에 한해서 명령을 실행한다.
- 대소문자 구분 잘하기

UNIX Programs

![https://s3-us-west-2.amazonaws.com/secure.notion-static.com/61089260-c1ee-446d-acbc-89e893ab4280/Untitled.png](https://s3-us-west-2.amazonaws.com/secure.notion-static.com/61089260-c1ee-446d-acbc-89e893ab4280/Untitled.png)

1. UNIT Process(중간에 원) : prcess 또는 program(ls, ps, mkdir 등과 같은 것) 
    - 프로그램 : 우리 컴퓨터 hdd, ssd에 저장된 코드 자체
    - 프로세스 : 그 프로그램이 실행되는 상태 (하나의 프로그램은 여러가지 프로세스를 가질 수 있음)
2. Command-line Arguments : 입력값 (ls -al에서 -al)
3. Standard Output(표준출력 1사용) : 프로그램 실행 결과값(ls라는 프로그램을 실행했을 때 나오는 결과값)
    - >라는 기호를 사용해서 결과값을 화면에 출력하는 것이 아니라 방향을 돌려서 파일에 Redirection 하여 저장 할 수 있다.
    - ls -l 1> result.txt    (꺽쇠 왼쪽에 1은 생략 가능)
4. Standard Error(표준에러 2사용) : Error 결과값 출력
    - rm result.txt 2> error.log   result.txt 삭제 명령에 대한 Error 내용을 error.log에 저장한다.
5. Standard Input (덜 중요) : 
    - cat < hello.txt
    - head -n1 < linux.txt  linux.txt의 첫줄만 출력(standard input)
    - head -n1 < linux.txt > one.txt   (그값을 one.txt에 redirection) (standard input, standard output 사용)

#### ls -l 1> result.txt 2> error.log   (1은 생략 가능)

#### ls-l 결과값(표준출력)이 있다면 result.txt로 redirection 될 것이고 만약 에러가 발생한다면 error.log로 저장될 것이다. (즉, 어떤 명령에 대한 결과값과 에러 모두 저장할 수 있다.)

#### 키보드로 입력해서 데이터가 INPUT, OUTPUT 되서 흐르는 방식을 IO STREAM이라 한다.
